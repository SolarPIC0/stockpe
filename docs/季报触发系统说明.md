# 季报触发系统说明

目标：在季报发布后，不只看“增长了多少”，而是判断这份季报是否足以迫使我们重算全年 EPS。

## 核心问题

一份好季报只有在改变全年盈利模型时，才可能成为机会。触发系统回答三个问题：

- 单季利润年化后，是否明显高于去年全年利润？
- 利润增速是否快于收入增速，说明毛利率、产品结构或经营杠杆改善？
- 这个增长是否有调研纪要、产能释放、客户认证、结构性需求作为前瞻支撑？

## 输入文件

默认输入：`data/quarter_trigger_input.csv`

关键字段：

- `last_year_net_profit`：上一年度归母净利润，单位保持一致，建议用亿元。
- `current_quarter_net_profit`：本季归母净利润。
- `current_quarter_net_profit_yoy_pct`：本季归母净利润同比增速。
- `current_quarter_revenue_yoy_pct`：本季收入同比增速。
- `current_quarter_deducted_net_profit`：本季扣非归母净利润。
- `current_quarter_operating_cash_flow`：本季经营现金流。
- `prior_year_q2_net_profit`：上年 Q2 单季归母净利润，用来推演“若 Q2 与 Q1 同利润，则 Q2 同比是多少”。
- `structural_driver_score`：0 到 2 分，AI、产能、客户认证、价格周期等结构性驱动。
- `research_signal_score`：0 到 2 分，机构调研、业绩说明会、IR 中是否出现可验证前瞻信号。
- `one_off_risk_score`：0 到 2 分，一次性收益、周期顶部、成本、现金流、竞争等风险。

## 输出解释

运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\quarter_trigger.ps1
```

可导出：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\quarter_trigger.ps1 -OutFile reports\季报触发系统输出_2026Q1.md
```

分数含义：

- `强触发：立即重算全年EPS`：季报可能改变全年盈利中枢，必须重新估值。
- `中强触发：进入重点跟踪`：有预期差线索，需要结合股价、调研、Q2 预告。
- `弱触发：等待下一项验证`：业绩不错，但证据不够强。
- `暂不行动`：没有足够信息证明盈利中枢上移。

## 使用纪律

- 这个工具不是买入信号，只是提醒“要不要重算估值”。
- 触发后必须再看股价是否已经反映、估值是否合理、Q2 基数是否抬高。
- 对周期股，PEG 只做辅助；真正关键是盈利平台是否持续抬升。
- 对 AI 链公司，调研纪要的价值在于发现产能、认证、订单和风险变化，不等于业绩承诺。
